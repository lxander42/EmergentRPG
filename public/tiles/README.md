# Tile atlas

`atlas.png` is a 256×48 sprite sheet of 16×16 frames, generated by
`scripts/generate-tiles.mjs`. Layout (col, row) → frame name lives in
`content/tiles.ts`.

The bundled atlas is a procedural placeholder drawn with the existing biome
and resource swatches so the sprite pipeline lands without an external asset
fetch. It mirrors the layout we'd use for a real Kenney pack: 16-wide grid,
16-px tiles, named frames keyed by role.

## Replacing with Kenney Tiny Town

Kenney's CC0 *Tiny Town* + *Tiny Dungeon* packs are the long-term target
(see issue #9). To swap in the real assets:

1. Download the packs from <https://kenney.nl/assets/tiny-town> and
   <https://kenney.nl/assets/tiny-dungeon>.
2. Either:
   - Re-export a single 256×48 atlas with the same frame layout listed in
     `content/tiles.ts` (easiest path — the manifest stays unchanged), or
   - Add new sheets and update `TILE_FRAMES` in `content/tiles.ts` to point
     at the new sheet/coordinate pairs. The loader in `lib/render/tiles.ts`
     already supports multiple sheet keys.
3. Drop the new PNG into `public/tiles/atlas.png` (or new sheets alongside).
4. Bump the version comment in `content/tiles.ts` so cached builds invalidate.

Both Kenney packs are CC0 — you can redistribute them in this repo without
attribution, though crediting Kenney in the project README is polite.

## Regenerating the placeholder

```bash
node scripts/generate-tiles.mjs
```

Edits to the generator should preserve the frame index order — the manifest
is index-driven, not name-driven, so a re-ordering breaks every lookup.
